## Назначение приложения

Приложение предназначено для автоматизированной очистки файлов в MinIO, базах `Jaga-Articles` и `Jaga-Tasks` для удаляемых проектов. Оно проверяет наличие файлов по путям, полученным из результатов SQL-запросов к этим базам, и удаляет те, которые больше не нужны.

### Основные функции:
- Выполнение параметризованного SQL-запроса с пагинацией к базе данных (`jaga-tasks`, `jaga-articles`) для получения списка путей к файлам.
- Проверка существования файлов в MinIO по полученным путям.
- Удаление неиспользуемых файлов из MinIO.
- Поддержка SSH-туннелирования для безопасного доступа к БД и MinIO.
- Гибкая конфигурация через переменные окружения и properties-файл.
- Логирование всех этапов обработки: проверка, удаление, ошибки.
- Защита от удаления файлов в определённых проектах через список игнорируемых `jaga_project_id`.

Работает с ресурсами приложения: `projectsAttachmentsQuery.sql`, `usersAttachmentsQuery.sql`, `projectsForDeleteQuery.sql`, `usersForDeleteQuery.sql`.  
Поддерживает пагинацию для безопасной обработки больших объёмов данных без перегрузки памяти.

---

### Клиент для работы с базами данных PostgreSQL

- Предоставляет соединения к двум базам: `jaga-tasks` и `jaga-articles`.
- Поддерживает подключение через SSH-туннель (опционально).
- При активации SSH создаёт локальный порт-форвардинг на указанный порт (`SSH_DB_LOCAL_PORT`), после чего подключается к `localhost`.
- Все соединения корректно закрываются, туннель завершается при остановке приложения.

### Клиент для работы с MinIO

- Если `SSH_ENABLED = true`, создаётся SSH-туннель к серверу MinIO с пробросом порта (`SSH_MINIO_LOCAL_PORT` → `MINIO_REMOTE_PORT`).
- Клиент подключается к `localhost:SSH_MINIO_LOCAL_PORT` при использовании туннеля, иначе — напрямую к `MINIO_ENDPOINT`.
- Автоматически закрывает соединение и удаляет правило порт-форвардинга при вызове `close()`.

## Конфигурация приложения

Приложение настраивается через **переменные окружения (ENV)** или файл `application.properties`.  
**Переменные окружения имеют приоритет** над значениями в properties.

### Базы данных

#### Общие параметры
| ENV / Property        | Описание                                 | Дефолтное значение |
|-----------------------|------------------------------------------|--------------------|
| `DB_HOST`             | Хост базы данных                         | `localhost`        |
| `DB_PORT`             | Порт базы данных                         | `5432`             |
| `DB_QUERY_PAGE_SIZE`  | Размер страницы данных загружаемых из БД | 40                 |

---

#### Параметры для `jaga-tasks`
| ENV / Property         | Описание                     | Дефолтное значение |
|------------------------|------------------------------|--------------------|
| `TASKS_DB_NAME`        | Имя базы данных задач        | `postgres`         |
| `TASKS_DB_USERNAME`    | Пользователь БД задач        | `postgres`         |
| `TASKS_DB_PASSWORD`    | Пароль БД задач              | `postgres`         |

---

#### Параметры для `jaga-articles`
| ENV / Property             | Описание                     | Дефолтное значение |
|----------------------------|------------------------------|--------------------|
| `ARTICLES_DB_NAME`         | Имя базы данных статей       | `kernel`           |
| `ARTICLES_DB_USERNAME`     | Пользователь БД статей       | `postgres`         |
| `ARTICLES_DB_PASSWORD`     | Пароль БД статей             | `postgres`         |
| `STORAGE_DB_NAME`          | Имя базы данных хранилища    | `storage`          |

---

### MinIO

| ENV / Property            | Описание                           | Дефолтное значение       |
|---------------------------|------------------------------------|--------------------------|
| `MINIO_ENDPOINT`          | URL MinIO (при прямом подключении) | `http://localhost:9000`  |
| `MINIO_ACCESS_KEY`        | Access Key для аутентификации      | `minioadmin`             |
| `MINIO_SECRET_KEY`        | Secret Key для аутентификации      | `minioadmin`             |
| `MINIO_BUCKET`            | Название целевого bucket           | `my-bucket`              |

> Примечание: если используется SSH-туннель, клиент подключается к `localhost:${SSH_MINIO_LOCAL_PORT}`.

---

### SSH-туннелирование

| ENV / Property               | Описание                                 | Дефолтное значение |
|------------------------------|------------------------------------------|--------------------|
| `SSH_ENABLED`                | Включить SSH-туннель                     | `false`            |
| `SSH_HOST`                   | Адрес SSH-сервера                        | `ssh.example.com`  |
| `SSH_PORT`                   | Порт SSH-сервера                         | `22`               |
| `SSH_USERNAME`               | Имя пользователя для SSH                 | `sshuser`          |
| `SSH_PASSWORD`               | Пароль пользователя для SSH              | `sshpass`          |
| `SSH_DB_LOCAL_PORT`          | Локальный порт для БД                    | `5432`             |
| `SSH_MINIO_LOCAL_PORT`       | Локальный порт для MinIO                 | `9001`             |

> При `SSH_ENABLED=true`:
> - БД доступна через `localhost:${SSH_DB_LOCAL_PORT}`
> - MinIO доступен через `localhost:${SSH_MINIO_LOCAL_PORT}`

---

### Список игнорируемых проектов

| ENV / Property                 | Описание                                             | Дефолтное значение |
|--------------------------------|------------------------------------------------------|--------------------|
| `IGNORING_JAGA_PROJECT_IDS`    | Список `jaga_project_id`, файлы которых не удаляются | `100`              |

Формат: строка с числами через запятую, например: `100,101,102`.

### Логирование

Приложение использует SLF4J + Logback.

---

### Безопасность

- Все учётные данные передаются через безопасные каналы (SSH, HTTPS).
- Пароли не логируются.
- При работе в продакшене рекомендуется использовать TLS для MinIO и SSH-ключи вместо паролей.

---

### Требования к инфраструктуре

- Java 17+
- Доступ к PostgreSQL (через SSH или напрямую)
- Доступ к MinIO (через SSH-туннель или напрямую)
- Разрешения на чтение из БД и удаление объектов в MinIO
